# 重构记录 — 2026-05-28

## 目标

提升项目可维护性：消除技术债务、模块化代码、建立回归测试。

## 重构线

### 1. `scripts/build_pinyin.js` — 提高可读性

**问题**：函数职责混杂（60 行验证/转换/过滤/组装挤在单个函数内）、常量散落、无错误处理。

**变更** (`36b6810`)：
- 提取 `CONFIG` 对象统一管理常量
- 拆分为 `toPinyinTokens` → `extractSyllables` → `assembleIndex` → `buildPinyinIndex` 编排调用
- `rebuildEmojiData` 重命名为 `enrichEmojiData`
- 提取 `addPinyinToEmoji`，`pinyinFields` 数组配置化
- `main()` 包裹 try-catch

### 2. 删除 `translate_emoji_data.py`，翻译逻辑合并到 `update_emoji.js`

**问题**：Python 脚本作为手动中间步骤，翻译策略是脆弱纯字符串替换（`man` → `男性` 会破坏 `mermaid`），且两脚本输出格式不一致（`const` vs `export const`）。

**变更** (`1f548bf`)：
- 四类翻译映射（肤色/性别/动作/角色）+ `subCategory` 兜底合并入 JS
- gender 翻译改用 `\b` 词边界正则，修复误伤
- `update_emoji.js` 输出格式统一为 `export const emojiData`
- 添加 `axios` 依赖到 `package.json`

### 3. `update_emoji.js` — 现代化语法

**变更** (`debc82e`)：
- `indexOf` → `includes`/`startsWith`（6 处）
- `String.fromCodePoint.apply` → spread（1 处）
- C 风格 `for` 循环 → `for...of`（6 处）
- `function` 表达式 → 箭头函数（3 处）
- 字符串拼接 → 模板字面量（2 处）

### 4. `build_pinyin.js` — 回归测试

**变更** (`cc47825`)：
- 新增 `scripts/build_pinyin.test.js`，11 条用例
- 9 组高风险样本：`笑哭了`、`睡着了`、`乐谱`、`音乐键盘`、`银行`、`重复按钮`、`旅行与地点`、`浅肤色`、`拿白手杖`
- 覆盖多音字、`ü→v` 归一化、非中文输入边界
- `build_pinyin.js` 添加 `module.exports` + `require.main === module` 守卫
- `npm test` 运行此测试集

### 5. 清理死代码

**变更** (`2950ea9`)：
- 删除 `static/js/vendor/tiny-pinyin.js`（17KB，已被 `pinyin-pro` 替代）
- 删除空 `tools/` 目录
- 同步更新 `readme.md` 项目结构

### 6. 搜索逻辑模块化 — `search.js`

**问题**：搜索打分、排序逻辑硬编码在 `main.js`（UI 层），不可测试，10 个魔法数字散落。

**变更** (`09e2722`)：
- 新增 `static/js/search.js` 纯函数模块，导出：
  - `normalize(text)` — 搜索文本归一化
  - `buildIndex(text, pinyinData)` — 建立文本 + 拼音索引
  - `buildSearchData(emoji)` — 为单个 emoji 建立完整搜索数据
  - `score(searchData, query)` — 加权打分
  - `search(items, term)` — 搜索排序
- 魔法数字提取为 `SCORE_BONUS`（14 个常量）和 `FIELD_WEIGHT`（4 个常量）
- `main.js` 350→232 行，仅保留 DOM 渲染和用户交互
- `index.html` 添加 `search.js` 的 `modulepreload`

### 7. 分离数据层与渲染层

**问题**：`main.js` 中仍残留 `createSearchableEmojiData()` 数据转换函数。

**变更** (`d91f5e4`)：
- `indexEmojiData(data)` 移到 `search.js`
- `main.js` 仅 `import { indexEmojiData, search }` + 两行调用，不再包含任何数据转换函数定义

### 8. `search.js` — 单元测试

**变更** (`7daa48e`)：
- 新增 `static/js/search.test.js`，27 条用例
- 回归基线锁定：评分权重、排序规则、短 ASCII 门控、交叉字段加分
- `npm test` 改为跑两组测试（`build_pinyin` + `search`），共 38 条，0 fail

## 最终架构

```
scripts/
├── build_pinyin.js          # 拼音索引生成
├── build_pinyin.test.js     # 拼音测试（11 条）
└── update_emoji.js          # emoji 数据获取 + 翻译

static/js/
├── search.js                # 数据层：纯函数（normalize / score / search / indexEmojiData）
├── search.test.js           # 搜索测试（27 条）
├── main.js                  # 渲染层：DOM 事件 + 分页 + 复制
└── emoji-data.js            # 静态数据（带拼音索引）
```

数据流：

```
update_emoji.js ──→ emoji-data.js ──→ build_pinyin.js ──→ emoji-data.js (带拼音)
                                            │
                                    emoji-data.js ──→ indexEmojiData() ──→ search() ──→ DOM
```
